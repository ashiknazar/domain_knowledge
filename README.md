# Domain Knowledge Repository

This repository is structured to help build strong domain understanding for Data Analyst, Data Scientist, ML/AI Engineer, and Big Data roles.

The goal is to organize domains first, and then incrementally add:

* Business understanding (KPIs, workflows)
* Data understanding (events, schemas)
* Technical implementations (pipelines, systems)

---

# Classification Approach

Each domain is explored using the following structure:

## 1. Business Understanding

* Key metrics (KPIs)
* Core workflows
* Decision-making logic

## 2. Data Understanding

* Data sources
* Event types
* Schema and attributes

## 3. Technical Implementation

* Data pipelines
* Processing frameworks
* Storage and querying systems

# How to Use This Repository

1. Start with 1–2 core domains (e.g., ecommerce, media)
2. Expand into subdomains gradually
3. For each domain, create notes under:

   * business.md
   * data.md
   * system.md
